# Impeccable — Design Context for Loop In

## Design Context

### Users
Two audiences. **Builders / founders / PMs** (the admins) who triage feedback, run a
roadmap, and ship — they judge the product on whether it feels precise and trustworthy.
And **end users / community members** who submit and upvote ideas — they need it to feel
welcoming and alive. This is also a **portfolio piece**: the real third audience is
**hiring managers** evaluating craft. The landing page must make a designer/engineer
look hireable in 5 seconds.

### Brand Personality
**Editorial, warm, deliberate.** Voice is confident and human, never corporate. Tagline:
"Close the loop with your users." The product idea is a *loop* — feedback that goes
somewhere, nobody shouts into the void.

### Aesthetic Direction
**One global identity: "Signal Stark"** (global as of 2026-06-17 — previously the warm
app chrome and the stark landing were split; the warm Fraunces-serif treatment is retired).

- **Stark monochrome canvas** — near-white `#F4F4F1` / near-black `#0B0B0C`, neutral grays —
  with **amber as the single accent**. Editorial: asymmetric layouts, **hairline rules over
  boxes/shadows**, index/ghost numbers, mono labels. **Oversized bold grotesque** (Space
  Grotesk display, NOT serif) carries every headline. Full light + dark, class-based.
- **Marketing landing (`/` logged out) + signed-in workspace home: cinematic stark.**
  Retuned to a reference GIF (`qclay.design`, an Awwwards agency site): full-viewport beats,
  **Lenis momentum scroll**, panel reveals, a rotating "scroll" **sticker badge**, an
  **animated hero graphic** (interlocking-loop mark with orbiting amber signal dots), an
  **interactive cursor** (amber dot + trailing ring; fine-pointer only), **ghost/outline
  numbers**, a **lifecycle word-highlight** list (Submit → … → Ship, each word lights amber
  on scroll), and one dark full-viewport **CTA beat**.
- **App chrome (Board / Admin / Thread / AppShell):** same stark canvas + grotesque + mono,
  but "tasteful app" — micro-interactions only (hover lifts, upvote pop, drag spring), no
  full-screen cinematics. See `docs/DESIGN-SYSTEM.md` → "Landing — Signal Stark".

The gold→slate-blue gradient is reserved for the **logo mark only**. Amber is the *only* UI
accent — the reference's violet/indigo→pink is recolored to amber.

Hard rule: **no violet / indigo→pink / teal** (reads AI-generated; rejected).

**Anti-references (explicitly banned — these read "AI-generated"):**
- Tinted-circle feature icons in a 3-column card grid.
- Centered hero + centered CTA band (everything symmetric).
- "Everything you need" bento grids.
- Uniform soft drop-shadows + same border-radius on every surface.
- Violet / indigo→pink / teal accents.

### Design Principles
1. **Type carries the page.** Dramatic scale contrast — bold grotesque (Space Grotesk)
   everywhere for display; mono labels with index numbers do the wayfinding. If a section
   feels flat, make the type bigger and the rule thinner — don't add a box.
2. **Asymmetry over symmetry.** Content sits off-grid, hangs into margins, breaks columns.
   Avoid dead-center compositions.
3. **Rules, not boxes.** Structure with hairline borders and whitespace, not cards with
   shadows. Sharp, flat, deliberate surfaces; minimal radius.
4. **Amber is a signal, not decoration.** One accent, used where it means something
   (CTA, active, upvote, an underline). Everything else is ink/paper/slate.
5. **Motion is editorial.** Rules draw in, type rises behind a mask, numbers settle, the
   loop mark draws on. GSAP for the landing showpiece; never a decorative gradient blob.
6. **Show the real product.** Prefer an honest framed product shot (figure + caption) over
   abstract illustration.
